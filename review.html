In making your own website, are you happy with the tools you used?

For my website I mainly used Visual Studio Code,Trello and Git, for both the implementation and organisation of my website. Both IT tools were very useful to me but obviously each had individual drawbacks I adapted to and changed to ensure the website would run as smoothly as possible. First of all, the use of VSC was very useful and I am very happy with the overall performance. It was very user friendly and not too hard to grasp for beginners, command lines were simple and easy to remember and most issues came from personal formatting issues that I was able to spot and change. There was a wealth of resources and help online if I had any queries about html or css and how to implement features that I wanted for my website. At first I had most of my code in html but once I learned how to link a css file I formatted my website to add features into the css file and make it much easier to organise and read. Overall I am very happy with VSC.

Furthermore I used trello to organise my website, admittedly I did not use trello for the beginning of my website but as I had more and more tasks to keep track of it worked very well for me to organise my workload. At first, I believed it wasn't very useful to me but this changed as my workflow expanded and I realised how useful trello could be even on your own. Finally I used Git to log my changes to my website. At first, this was relatively tricky to get to grips with all the appropriate commands but once I had “Git add” “Git status” and “Git commit” several times, it became a lot easier. Git was very useful for everyone who needs to access and see the progress of my website, at the end of the day I can clearly see my logged changes and how my website has progressed that day. Overall, i am very happy with Git.

Assess the strengths and weaknesses of your final project

Overall, i’d say i am very happy with my website, but no project comes without criticism and something can always be improved. For my first website I thought I performed well with my very limited experience. The strengths of my website in my own opinion is the Graphical User Interface. I believe I did the appropriate extra research into making my website as presentable as possible. Even though it is at a relatively low level, even taking the extra time to research how to have an image as a background instead of a block colour really helped the overall professional look of my website, and being able to complete the project on time meant I had some spare time to do extra research into very basic animation, which I used to animate a ship on my welcome page. I tried to keep the look immersive but not cluttered, so i designed nice clean buttons that matched the style of the website but also helped me break up and connect my website into pages. This leads me onto the another strength of the website and that's cleanly represented data. It wasn't until I had a million tasks to keep track of that I realised how useful Trello was, this useful learning curve will serve me well in the future though. In addition, a weakness of my website is the accessibility, if i wanted to make this website more professional i would be adding accessible features for disabled users as previously explored in a question. Also, even though I think I kept my website pretty clean, the gifs implemented might not be to the taste of everyone, and a weakness of the project could be seen as distracting images, however I am pretty confident it still remains clean and readable in most aspects.

In conclusion, Id say my website has stronger strengths than weakness’ however It would be stupid to believe the website is free from weakness and even I could improve it if I were to start the website again. However, overall I am very proud of my first website and look forward to access the strengths and weakness’ of many more to come.

Describe which are the appropriate file types for websites

The two files types we used with Visual Studio Code were HTML and CSS. Each file type had an important role to play and work cohesively to deliver a refined overall product. HTML is the language for basic control of a website and describing the data of a website. HTML gives the user the ability to Publish online documents with headings, text, tables, lists, photos, etc. In addition, it gives the user the ability to Retrieve online information via hypertext links, at the click of a button. Design forms for conducting transactions with remote services, for use in searching for information, making reservations, ordering products, etc. And finally, Include spread-sheets, video clips, sound clips, and other applications directly in their documents. Basically the HTML controls all aspects of the physical data of the website and how to organise it relatively primitively.

Furthermore, this leads onto the use of CSS. “CSS is the language for describing the presentation of Web pages, including colors, layout, and fonts. It allows one to adapt the presentation to different types of devices, such as large screens, small screens, or printers. CSS is independent of HTML and can be used with any XML-based markup language. The separation of HTML from CSS makes it easier to maintain sites, share style sheets across pages, and tailor pages to different environments. This is referred to as the separation of structure (or: content) from presentation.”

In conclusion, HTML works well to input data and CSS compliments this by structuring and formatting the HTML data into a user friendly and attracitve website.
